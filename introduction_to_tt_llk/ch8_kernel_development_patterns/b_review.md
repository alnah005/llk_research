# Agent B Review: Chapter 8 — Kernel Development Patterns — Pass 1

1. **Supported tile sizes list is incomplete (eltwise_binary_walkthrough.md, "Tile Dimensions" section).** The text states the supported tile sizes come from `SUPPORTED_TILE_SIZES` and lists `32x32`, `32x16`, `16x32`, and `16x16`. The actual `SUPPORTED_TILE_SIZES` in `tests/python_tests/helpers/tile_constants.py` also includes `1x32`, `2x32`, `4x32`, and `8x32`. A reader implementing or extending tests would miss half the valid tile configurations.

2. **Format space omits Bfp4_b (eltwise_binary_walkthrough.md, "Format Parametrization" section).** The text says "The format space includes `Bfp8_b`, `Float16_b`, and `Float32`." The actual `_get_valid_formats` function in `test_zzz_eltwise_binary.py` also includes `Bfp4_b` (line 78). The fidelity mapping text similarly says "Bfp8 supports only LoFi" without mentioning that `Bfp4_b` has the same constraint. A reader would not know `Bfp4_b` is a valid and tested format for eltwise binary operations.

3. **Matmul ADDR_MOD detailed table is missing ADDR_MOD_4 (matmul_walkthrough.md, "Address Modifier Configuration" section).** The summary comparison table at the end claims matmul uses "7 slots (ADDR_MOD_0-6)", but the detailed table in the Address Modifier Configuration section lists only 6 entries (0, 1, 2, 3, 5, 6) and omits ADDR_MOD_4 entirely. In the source (`llk_math_matmul.h`), ADDR_MOD_4 is used for 16x32 and 32x16 tile size variants. A reader implementing support for non-32x32 tile sizes would not know ADDR_MOD_4 exists or what it does.

4. **Matmul walkthrough omits `_llk_math_pack_sync_init_` and `_llk_math_hw_configure_` from the MATH section (matmul_walkthrough.md).** In the actual `matmul_test.cpp`, the math section calls `_llk_math_matmul_init_` (line 72), then `_llk_math_pack_sync_init_` (line 73), then `_llk_math_hw_configure_` (line 74). The walkthrough's MATH section only discusses `_llk_math_matmul_init_` and then jumps to the compute loop, omitting the two required setup calls. A reader writing a matmul kernel from this walkthrough alone would produce code that fails at runtime due to missing sync and hardware configuration.
